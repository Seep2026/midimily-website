# Narrative Review Checklist

本清单用于每个方案发布前的叙事审查。  
建议在 `slides.md` 完成后逐项检查，全部通过再发布。

## A. 受众与场景

- [ ] 目标用户是否明确（不是“所有人”）？
- [ ] 用户场景是否真实可感（不是抽象概念）？
- [ ] 问题是否来自真实工作/学习流程，而非想象痛点？

## B. 核心判断与逻辑

- [ ] 核心判断是否一句话讲清？
- [ ] 是否解释“为什么很多人做了但没结果”？
- [ ] 是否给出从现状到目标的路径，而不只给结论？

## C. 结构与节奏

- [ ] 是否包含行动路径（步骤/层次/阶段）？
- [ ] 是否包含可执行行动建议（如 7 天/30 天第一步）？
- [ ] 是否做到每页只讲一个重点？
- [ ] 是否避免长段落，改为短句与要点？

## D. 语气与品牌一致性

- [ ] 是否避免自吹自擂与夸张承诺？
- [ ] 是否保持清晰、温和、商务、可信语气？
- [ ] 是否避免“焦虑营销”“神化 AI”表述？

## E. 结尾与行动

- [ ] CTA 是否自然衔接内容？
- [ ] CTA 是否只有一个主动作？
- [ ] 读者读完后是否知道“下一步做什么”？

## 审查结论

- [ ] 通过，可发布
- [ ] 不通过，需返工（标注返工页码与原因）
