# MIDIMILY Deck Design System

本文件定义 midimily Web Deck 的视觉和表达基线。  
目标：保证不同方案由 Codex 生成时，风格稳定、阅读顺滑、行动导向明确。

## 1) 品牌气质

- 轻科技：结构清晰、信息密度适中、视觉克制。
- 温馨商务：可信、专业、可沟通，不冷硬。
- 以“问题-路径-行动”为中心，不以炫技为中心。

## 2) 色彩与界面

- 主背景：暖白（如 `#fcf8f2` 附近）
- 辅助背景：浅蓝灰（如 `#f7f9fc` 附近）
- 强调色：低饱和蓝紫（如 `#7c92bb` 附近）
- 辅助强调：浅青（如 `#8cc7bd` 附近）

使用原则：

- 大面积使用浅色，强调色仅用于关键按钮、状态、引导。
- 不使用高对比荧光色，不使用霓虹紫蓝渐变。
- 阴影保持轻微，避免重阴影和“漂浮卡片堆叠感”。

## 3) 版式与信息节奏

- 每页只讲一个重点。
- 中文短句优先，避免长段落。
- 一页要点建议 3-4 条，单条尽量短。
- 标题可判断、可执行，不写空泛口号。

## 4) 文案语气

- 清晰、温和、商务。
- 问题真实，结论可落地。
- 不自吹自擂，不夸张承诺，不制造焦虑。
- 避免“万能解法”语气，强调“先小闭环再扩展”。

## 5) 视觉禁区

- 不做赛博朋克风格。
- 不做重动画、炫技转场、过度粒子效果。
- 不做高频闪烁或干扰阅读的动效。
- 不堆叠大量装饰图形抢占主信息。

## 6) CTA 规则

- CTA 必须自然衔接前文，不突兀销售化。
- CTA 用“下一步动作”表达，如“预约咨询”“开始梳理”。
- CTA 只保留 1 个主动作，避免多按钮分散注意力。

## 7) 交付标准

- 可在 3-5 分钟内快速读完并形成行动判断。
- 首次阅读者可明确回答：
  - 我的问题是什么？
  - 为什么现在要做？
  - 第一小步该做什么？
