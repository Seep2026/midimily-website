# 在 AI Agent 元年，<br />全面理解 AI Agent

从大模型、AI 工作流，到真正能完成任务的智能体

> 大模型回答问题，AI Agent 完成任务。

# AI 正在从“会聊天”走向“会办事”

- 很多人已经会用 ChatGPT、Claude、Gemini
- Agent 不是更会聊天的机器人
- Agent 代表 AI 从回答问题，走向执行任务
- 个人要理解的是一种新工作方式

> Agent 的出现，代表 AI 从“会说”走向“会做”。

# 第一层：大模型是会回答问题的大脑

- 生成文字、修改内容
- 总结资料、写邮件
- 写代码、头脑风暴
- 但它默认不知道你的真实世界
- 它通常要等你提问才回答

> 大模型很聪明，但默认不会主动办事。

# 第二层：AI 工作流会按预设路径执行

- 用户问下一次会议时间
- 系统先查 Google Calendar
- 再让大模型组织回答
- 如果没接天气 API，它就不会查天气

> Workflow 很有用，但它只能沿着人类预设的路径走。

# 第三层：Agent 的关键，是 AI 开始做决策

- 人给目标
- AI 判断路径
- AI 选择工具
- AI 执行任务
- AI 根据结果调整

> AI Agent 不是工具更多，而是大模型开始决定下一步怎么做。

# Agent 的工作方式：想一想，做一步，看结果

- Reason：理解目标，思考下一步
- Act：调用工具，执行动作
- Observe：观察结果，判断是否调整
- 不符合标准，就继续迭代

> Agent 的能力，不只在执行，也在反馈后的修正。

# 一个 Agent 通常由这些部分组成

- LLM：理解、推理、生成
- Instructions：角色、目标、边界
- Tools：浏览器、API、文件、代码执行器
- Memory：偏好、上下文、任务状态
- Planner / Executor：拆解并执行步骤
- Feedback Loop：根据结果修正

> Agent 不只是模型，而是围绕目标运行的小系统。

# 从“每天自动发帖”到“自己优化内容”

- Workflow：按固定路径自动运行
- Agent：自己判断该选哪些新闻
- 自己生成不同版本
- 自己调用模型评审
- 自己迭代到符合标准

> Workflow 是自动执行，Agent 是自动判断、执行和迭代。

# 为什么编程 Agent 最先成熟

- 能读取代码库
- 能拆解开发任务
- 能修改文件并运行命令
- 能执行测试并修复错误
- 代码任务结构清楚，反馈明确

> 编程 Agent 不是帮你写几行代码，而是在尝试接管一部分工程任务。
