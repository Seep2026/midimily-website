# Midimily UI/UX Design System

本文档是 midimily.com 后续 UI/UX 修改的主设计系统。任何由 Codex 或人工执行的视觉、交互、组件、页面、Web Deck 更新，都应先读取本文件，再进入具体实现。

## 1. 品牌关键词

米地米立 / Midimily 的品牌定位是：

> 企业的 AI 落地伙伴，个体的 AI 成长顾问。

核心关键词：

- 轻科技：有 AI 与系统感，但不做炫技视觉。
- 温馨商务：专业可信，同时有人情味和陪伴感。
- 清晰克制：把复杂问题讲清楚，不堆装饰。
- 可行动：每个页面都要帮助用户理解下一步。
- 长期陪跑：不是一次性咨询销售，而是持续成长伙伴。
- 双服务线：企业 AI 落地，个体 AI 成长。

品牌语气：

- 中文短句优先。
- 语气清醒、温和、有现实感。
- 不制造焦虑，不自吹自擂。
- 少用抽象大词，多讲场景、路径、流程、行动。

## 2. 视觉原则

整体视觉方向：

- 暖白背景作为主基底。
- 浅蓝灰建立专业可信感。
- 低饱和蓝紫表达 AI、认知、系统感。
- 浅青作为陪伴、成长、流动感点缀。
- 卡片和分区要轻，不做重阴影和厚重边框。
- 页面要有空气感，避免信息挤压。

设计判断：

- Midimily 不是赛博科技公司，不使用强霓虹、黑底蓝光、重网格。
- Midimily 不是传统咨询公司，不使用深蓝金色、商务图库、严肃模板。
- Midimily 不是模板化 SaaS，不使用大面积紫色渐变、浮夸 hero、无意义 dashboard 插画。
- Midimily 的高级感来自留白、层次、文案清晰和交互克制。

- 主页面不要被单一蓝色或紫色淹没。
- 企业 AI 落地方向可偏浅蓝紫。
- 个体 AI 成长方向可偏浅青蓝。
- CTA 使用蓝灰主色，不用高饱和红、橙、荧光绿。
- 时间戳、辅助版本号应接近水印，不抢封面标题。

## 4. 字体层级

字体原则：

- 主站和 Deck 都优先使用系统无衬线中文字体。
- 不引入花体、手写体、过强个性的字体。
- 当前建议字体栈：
  `Inter, "PingFang SC", "Microsoft YaHei", sans-serif`

主站层级：

| 层级 | 桌面端 | 移动端 | 用途 |
| --- | --- | --- | --- |
| Hero H1 | 52-68px | 34-42px | 首页主判断 |
| Page H1 | 44-56px | 32-38px | 方案库、主要页面 |
| Section H2 | 30-40px | 26-32px | 模块标题 |
| Card Title | 20-28px | 18-22px | 服务卡、方案入口 |
| Body | 15-18px | 15-16px | 正文说明 |
| Meta | 12-14px | 12-13px | 标签、时间、辅助说明 |

Web Deck 层级：

- 每页主标题固定从左上方开始排列。
- 标题区域可占面板宽度约 70%-75%。
- 内容区根据内容量自动居中，但不要挤压标题。
- 每页只突出一个主判断。
- 标题不应被动态垂直居中到页面中央，除非是封面页或强声明页。

## 5. 页面间距系统

基础间距：

- `4px`：细微对齐和图标间距。
- `8px`：标签、按钮内部元素间距。
- `12px`：小组件、气泡内容间距。
- `16px`：卡片内部紧凑间距。
- `24px`：卡片默认内边距、小模块间距。
- `32px`：模块内部大间距。
- `48px`：页面区块间距。
- `72px`：Hero 与下一模块的桌面端间距。
- `96px`：大型页面上下留白上限。

容器宽度：

- 主站内容最大宽度：`1120px` - `1220px`。
- 方案库封面墙最大宽度：`1220px`。
- Web Deck 舞台最大宽度：`1240px` 左右。

规则：

- 固定 Header 下方必须留出足够安全距离。
- 移动端横向 padding 建议 `20px` 或 `24px`。
- 页面不能出现横向滚动。
- 模块删除或新增后，必须检查上下衔接，不留明显断层。

## 6. 卡片规则

卡片用于：

- 方案封面。
- 服务方向入口。
- 实践案例。
- 联系方式。
- 宠物气泡。
- Deck 内部内容块。

卡片风格：

- 圆角通常为 `12px` - `18px`。
- 方案封面可以更大圆角，但不超过 `24px`。
- 边框使用低对比蓝灰线。
- 阴影极轻，优先使用边框和背景层级。
- 不要卡片套卡片，除非是 Deck 内部内容结构且视觉足够轻。

卡片交互：

- 可点击卡片必须有 `cursor: pointer`。
- Hover 可使用边框加深、轻微上浮、背景微变。
- Hover 不应导致布局尺寸变化。
- 过渡时间控制在 `150ms` - `300ms`。

方案封面卡：

- 只展示 Deck 封面预览和必要时间水印。
- 不重复标题、描述、标签、页数。
- 整张卡片点击进入 Web Deck。
- 时间戳位于右下角，低亮度、低饱和，像水印。

## 7. 按钮规则

按钮类型：

- Primary：主要行动，如预约咨询、打开 Web Deck。
- Secondary：返回、查看全部方案、弱 CTA。
- Ghost：导航、轻量入口。

尺寸：

- 桌面端按钮高度建议 `44px` - `52px`。
- 移动端点击区域不得小于 `44px`。
- 圆角 `12px` - `16px`，不使用胶囊按钮作为默认样式。

文案：

- 使用动词开头。
- 不写“立即赋能”“开启未来”等空泛表达。
- 推荐：
  - 预约企业 AI 咨询
  - 预约个体成长咨询
  - 查看全部方案
  - 返回
  - 打开 Web Deck

禁用：

- 不在公开页面显示 PDF 下载。
- 不在公开页面显示 PPT 下载。
- 不在公开页面显示导出按钮。
- 不增加登录、后台、CMS 入口。

## 8. 背景图案规则

允许：

- 暖白到浅蓝灰的低对比渐变。
- 极轻的节点、线条、网格、圆弧。
- 少量低饱和蓝紫和浅青光感。
- Deck 中可使用类似杂志化的留白、标题排布、分栏节奏。

限制：

- 背景图案透明度要低，不抢正文。
- 节点线条只用于暗示系统与连接，不做复杂网络海报。
- 不使用离散大光球、bokeh blobs、强发光装饰。
- 不使用纯 SVG 插画作为 Hero 主视觉替代真实内容。

Web Deck 背景：

- Deck 面板内部不应有厚重白边。
- 底板要和主站暖白背景融为一体。
- 不使用黑边播放器感。
- 不要把 Slidev 原生工具栏暴露为主体验。

## 9. 图标规则

图标风格：

- 优先使用一致线性图标。
- 推荐 Lucide 风格：线性、轻量、圆角端点。
- 图标尺寸统一，常用 `18px`、`20px`、`24px`。
- 图标颜色使用蓝灰或当前方向强调色。

禁止：

- 不用 emoji 作为 UI 图标。
- 不混用多套图标风格。
- 不使用高饱和彩色图标堆叠。
- 不用图标替代必须清楚表达的文字。

## 10. 首页规则

首页目标：

- 让用户快速理解 Midimily 做什么。
- 明确两条服务线：企业 AI 落地、个体 AI 成长。
- 展示实践与方案，不做自我介绍堆叠。

首页结构原则：

- 首屏要直接呈现定位，不做纯营销 Hero。
- 企业服务与个体成长要并列清楚。
- 方案预览应进入 Web Deck，不进入冗余详情页。
- 不再出现“观点”字样。
- 不再放“为什么是米地米立”式自我介绍模块。

首页视觉：

- 信息密度适中，避免落入 SaaS 模板 hero。
- 卡片数量不要过多。
- CTA 直接、低压、不夸张。
- 宠物形象可以出现，但不能遮挡主要 CTA。

## 11. 方案库规则

方案库目标：

- 像“方案封面墙”，不是博客列表。
- 强化 Web Deck 作为主体验。
- 支持企业与个体两类方向筛选。

方案库 Hero：

- 标题短，建议“精选 AI 方案”。
- 副标题说明 Web Deck 形式即可，不把技术词作为理解门槛。

分类筛选：

- 只显示“企业 AI 落地方案”和“个体 AI 成长方案”两个方向卡。
- 默认展示全部。
- 点击已选方向可取消筛选，恢复全部。
- 选中态用轻边框、浅底、右上角呼吸点。
- 不显示“全部方案”按钮。
- 不显示“当前选择”文字标签。

方案封面：

- 桌面两列，移动单列。
- 封面内只展示视觉化标题、副标题、时间水印。
- 右下角时间只显示年月日，暗到像水印。
- 按更新时间倒序排列。
- 点击直接进入 `/solutions/{slug}/slidev/`。

## 12. Web Deck 规则

Web Deck 是方案的主体验。

页面结构：

- 主站 Header 保留。
- Deck 主体居中。
- 页尾可按页面整体需要出现，但不要干扰阅读。
- 返回按钮文案使用“返回”。
- 页码快捷按钮使用数字 + “页”开关。

交互：

- 点击当前页面可进入下一页。
- 数字页按钮用于跳转指定页。
- 点击数字页后，页码面板不自动消失；再次点击“页”才收起。
- 支持键盘左右键。
- 不显示上一页 / 下一页文字按钮作为主控。
- 不显示新窗口打开。
- 不显示进度条或页码大标题，除非后续明确需要。

版式：

- 每页一个核心判断。
- 标题固定左上方。
- 内容区根据内容量自适应排布。
- 内容多时自动降低字体和卡片密度，不能溢出面板。
- 章节页要有空间感，不要把所有内容塞满。
- 企业方案与个体方案风格一致，但可通过色彩倾向区分。

文案：

- Deck 不是文章页。
- 每页标题尽量不超过 24 个汉字。
- 每页要点尽量不超过 4 条。
- 每条要点尽量不超过 18 个汉字。
- 最后一页自然 CTA，不硬推销。

## 13. 移动端规则

移动端目标：

- 不遮挡，不横滚，可读，可点。

规则：

- Header 可折叠菜单，导航不拥挤。
- 所有按钮点击区域至少 `44px`。
- 卡片单列。
- 方案封面比例可比 16:9 更高，保证中文可读。
- Web Deck 不强制桌面比例；内容可纵向更舒展。
- 宠物默认隐藏或仅保留低干扰入口。
- 不允许 fixed 元素遮挡 CTA、联系二维码、方案翻页控件。

测试断点：

- `375px`：小屏手机。
- `768px`：平板 / 大屏手机。
- `1024px`：小桌面。
- `1440px`：常规桌面。

## 14. 常驻宠物形象规则

当前品牌宠物：

- 名称：小米立。
- 角色：轻量 AI 引导助手。
- 位置：右下角，桌面常驻，移动端低干扰。

行为：

- 动作自然，不能长期重复单一动作。
- 不遮挡主要按钮、方案封面、联系区。
- 气泡提示不频繁弹出。
- 引导按钮跳转到主页对应锚点时必须正确工作。

语气：

- 温和、陪伴、轻提醒。
- 不像客服机器人。
- 不使用“主人”“命令”等不合适表达。

## 15. 禁止项

视觉禁止：

- 赛博朋克。
- 强霓虹。
- 黑金风。
- 传统咨询公司深色商务模板。
- 大面积紫粉渐变。
- 高饱和蓝紫主导全站。
- 重阴影、厚边框、强玻璃拟态。
- 无意义大图、模糊 stock 图。
- 装饰性渐变圆球、bokeh blob。

内容禁止：

- 自吹自擂。
- 制造 AI 焦虑。
- 空喊“赋能”“颠覆”“重塑未来”。
- 把方案写成普通长文章。
- 在公开页面提供 PDF/PPT 下载 UI。
- 增加后台、CMS、登录、导出入口。

交互禁止：

- Hover 导致布局抖动。
- 固定元素遮挡正文或 CTA。
- 移动端横向滚动。
- Deck 翻页后内容不更新。
- 外层页面和内嵌 Deck 出现重复 Header。

## 16. Codex UI 修改 Checklist

每次 Codex 后续改 UI 前必须检查：

- [ ] 已读取 `design-system/MIDIMILY_MASTER.md`。
- [ ] 未引入赛博、强霓虹、黑金、传统咨询风。
- [ ] 页面仍然服务“企业 AI 落地 / 个体 AI 成长”双主线。
- [ ] 文案清晰、温和、可行动，没有自吹自擂。
- [ ] 色彩使用暖白、浅蓝灰、低饱和蓝紫、浅青。
- [ ] 字体使用系统无衬线，主站和 Web Deck 保持一致。
- [ ] 卡片不嵌套卡片，不使用重阴影。
- [ ] 所有按钮点击区域至少 `44px`。
- [ ] 可点击卡片有 hover、focus、cursor 反馈。
- [ ] 移动端无横向滚动。
- [ ] Header 固定时，内容没有被遮挡。
- [ ] 方案库封面只保留封面预览和水印级时间。
- [ ] Web Deck 无 PDF/PPT/导出/登录入口。
- [ ] Web Deck 标题固定左上，内容不溢出面板。
- [ ] 宠物不遮挡关键操作。
- [ ] 重要页面在 375px、768px、1024px、1440px 检查过。
- [ ] `pnpm lint` 和 `pnpm build` 在需要时通过。
